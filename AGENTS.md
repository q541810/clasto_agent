# Clasto Agent - 项目注解

## 项目概述
Clasto Agent 是一个 AI 代理网关，通过 WebSocket 接收适配器消息，与 AI 模型交互后回复。

## 架构
```
QQ用户 → Napcat → Napcat适配器(正向WS) → 主程序(WS:8081) → 聊天管理器 → 规划模型 → 工具执行 → 回复模型 → 回复回传
```

## 核心模块
| 文件 | 说明 |
|------|------|
| `main.py` | 主入口，WS服务端(localhost:8081)，消息队列调度，API等待池，适配器API调用，插件系统初始化，配置验证 |
| `模块/聊天管理器.py` | 按群/私聊管理独立聊天流，维护对话历史，规划模型+工具执行+回复模型三阶段处理，各阶段触发插件钩子 |
| `模块/工具执行器.py` | 工具注册表，内置工具(回复/发送消息/查询群列表/获取当前时间/联网搜索/获取会话消息)，并根据工具注册元信息自动生成规划模型system prompt(含参数说明与示例)，支持插件注册自定义工具 |
| `模块/插件管理器.py` | 插件系统核心：扫描/加载/管理插件，钩子注册与调度，插件上下文对象 |
| `模块/配置管理器.py` | 统一配置管理：单例模式，配置缓存，提供标准的配置获取接口 |
| `模块/配置验证器.py` | 配置验证：启动时验证配置完整性，检查API Key、模型名称等必填项 |
| `模块/openai格式模型调用.py` | OpenAI兼容格式的模型调用封装 |
| `模块/reading_config.py` | 配置读取（已废弃，保留用于向后兼容），建议使用配置管理器 |
| `模块/启动适配器.py` | 扫描并启动适配器进程 |
| `模块/适配器/napcat/` | Napcat适配器，支持正向/反向WS，支持call_api action |
| `模块/适配器/已废弃/` | 已废弃的适配器存放目录 |

## 配置文件
| 文件 | 说明 |
|------|------|
| `配置文件/api_config.toml` | API厂商和模型配置 |
| `配置文件/model_config.toml` | 回复模型配置(模型名、system prompt、历史长度、额外参数) + 规划模型配置(模型名、system prompt、额外参数) + 筛选模型配置(模型名、system prompt、额外参数) |
| `配置文件/runtime_config.toml` | 运行时逻辑配置（当前含bot名字、模型调用自动重试次数、消息分割器开关、群聊/私聊独立回复逻辑：直接规划或群聊回复优化、@必回复、提及关键词、关闭规划模型；并单独提供群聊回复优化参数如上下文条数；后续可扩展更多主程序配置） |
| `模块/适配器/napcat/config.toml` | Napcat连接配置、会话筛选(群/私聊独立白/黑名单) |
| `配置文件模板/plugin_config.toml` | 插件系统全局配置模板 |

## 消息流转
1. 适配器接收消息 → WS发送到主程序 `消息队列`
2. 定时批处理(1s) → `待处理队列`
3. 解析JSON → **`on_message_receive` 钩子** → `聊天管理器.分发消息()` → **`on_message_enqueue` 钩子** → 路由到对应聊天流
4. 聊天流处理：
   - 加入用户消息到对话历史
   - **群聊回复优化（可选）**：按 `runtime_config.toml` 对群聊/私聊分别配置；先由筛选模型输出简短原因和"回复/不回复"决策
   - **规划阶段**：**`on_pre_plan` 钩子** → 调用规划模型分析消息，输出JSON工具调用列表 → **`on_post_plan` 钩子** → 解析工具调用
   - **工具执行**：先执行非回复工具(如发送消息、查询群列表)，**`on_post_tool` 钩子** → 注入操作摘要到对话历史，再执行回复工具
   - **回复阶段**：**`on_pre_reply` 钩子** → 回复工具调用回复模型生成回复 → **`on_post_reply` 钩子** → **`on_send_message` 钩子** → 回复通过WS发回适配器

## 统一消息格式
```json
{"adapter":"napcat","chat_id":"group_123","chat_type":"group","sender_id":789,"sender_name":"用户","message":"内容","raw_data":{}}
```

## 主程序→适配器消息格式
```json
{"action":"send_message","chat_type":"group","chat_id":"group_123","message":"AI回复内容"}
```

## 主程序→适配器API调用格式
```json
{"action":"call_api","echo":"uuid","api_action":"get_group_list","api_params":{}}
```

## 适配器→主程序API响应格式
```json
{"status":"ok","retcode":0,"data":[...],"echo":"uuid"}
```

## 规划模型输出格式
```json
{"tool_calls": [{"name": "工具名", "parameters": {"参数名": "参数值"}}]}
```

## 规划提示词生成机制
- 规划模型的工具清单、参数说明、示例由 `模块/工具执行器.py` 根据工具注册元信息自动生成
- `配置文件/model_config.toml` 中的 `规划模型.system_prompt` 仅作为基础提示词前缀

## 内置工具
| 工具名 | 别名 | 说明 |
|--------|------|------|
| 回复工具 | 回复 | 进入回复阶段，调用回复模型生成回复；支持可选参数`备注`用于补充上下文 |
| 不回复工具 | 不回复/跳过回复 | 明确本轮不回复用户，可附带可选原因 |
| 发送消息工具 | 发送消息/发消息 | 直接向指定会话发送消息 |
| 查询群列表工具 | 查群/群列表 | 查询当前账号已加入的所有群聊 |
| 获取会话消息工具 | 获取会话消息/查询消息/消息记录 | 获取某个群聊或私聊最近消息，支持条数参数(默认5，上限30) |
| 获取当前时间工具 | 获取时间/查时间/时间 | 获取当前系统时间和星期 |
| 联网搜索工具 | 联网搜索/搜索/bing搜索 | 使用Bing公开搜索页进行联网搜索(无需API) |
| 多轮执行工具 | 多轮执行/继续规划/继续执行 | 请求进入下一轮规划，并携带备注等上下文信息 |

## 调试模式
启动时添加 `-debug` 参数即可启用调试模式，日志级别将切换为 DEBUG，并打印规划模型和回复模型的完整输入内容(system prompt + user prompt)：
```bash

```

## 消息分割器
在 `配置文件/runtime_config.toml` 的 `[消息分割器]` 中配置，默认关闭。

启用后，回复模型将使用流式输出（自动忽略额外参数中的 `stream` 配置并输出 warn 日志），并在输出遇到换行符 `\n` 或中文句号 `。` 时，将已累积的内容作为一条消息分段发送出去。

示例：模型输出内容为：
```
Ciallo～(∠・ω< )⌒☆ 主人主人，麦麦在这里喵！(≧∇≦)ﾉ

刚才麦麦正在舔毛，顺便在回味上次那个超级香的小鱼干呢，喵呜~
```

分割器行为（流式输出到分隔符时立即发送并清空缓冲，分隔符本身不包含在发送内容中）：
- 输出到第一行换行时，发送 `"Ciallo～(∠・ω< )⌒☆ 主人主人，麦麦在这里喵！(≧∇≦)ﾉ"`
- 输出到第二行结尾 `\n` 时，发送 `"刚才麦麦正在舔毛，顺便在回味上次那个超级香的小鱼干呢，喵呜~"`

配置示例：
```toml
["消息分割器"]
启用 = false  # 启用后回复模型将使用流式输出，并在输出到换行或句号时分段发送消息
```

## 插件系统

### 目录结构
```
插件/                          ← 插件目录（启动时扫描）
  我的插件/
    plugin.toml               ← 插件元信息 + 默认配置
    main.py                   ← 插件入口（必须包含 入口 函数）
插件配置/                      ← 用户配置覆写目录（可选）
  我的插件.toml               ← 覆写 plugin.toml 中的 config 部分
```

### plugin.toml 格式
```toml
[plugin]
name = "我的插件"
version = "1.0.0"
description = "示例插件"
author = "xxx"
priority = 100          # 钩子执行优先级，数字越小越先执行

[plugin.config]
# 插件自定义配置项，可被 插件配置/<插件名>.toml 覆写
api_key = ""
max_count = 5
```

### 插件入口函数
```python
# 插件/我的插件/main.py
async def 入口(上下文):
    """插件入口函数，启动时调用一次"""
    上下文.注册工具(名称="我的工具", 工具函数=我的工具函数, 别名=["别名"], 参数说明={"关键词": "搜索关键词"}, 说明="示例工具")
    上下文.注册钩子("on_post_reply", 我的钩子函数)
```

### 插件上下文 API
| 方法 | 说明 |
|------|------|
| `注册工具(名称, 工具函数, 别名, 参数说明, 说明, 是否回复工具)` | 注册工具到工具注册表，规划模型自动可获得 |
| `注册钩子(hook名, 回调)` | 注册生命周期钩子 |
| `调用适配器API(action, params)` | 调用适配器API（如 get_group_list 等） |
| `获取聊天流(chat_id)` | 获取指定聊天流的对话历史 |
| `发送消息(chat_id, chat_type, message)` | 直接发送消息到指定会话 |
| `获取配置(键=None)` | 读取插件配置（自动合并用户覆写） |

### Hook 列表
| Hook | 签名 | 返回值 | 类型 | 说明 |
|------|------|--------|------|------|
| `on_load` | `()` | 无 | 通知型 | 插件加载完成 |
| `on_unload` | `()` | 无 | 通知型 | 插件卸载 |
| `on_message_receive` | `(消息数据)` | `消息数据` 或 `None`(拦截) | 过滤型 | 适配器消息进入主程序时（main.py） |
| `on_message_enqueue` | `(消息数据)` | `消息数据` 或 `None`(拦截) | 过滤型 | 消息进入聊天流队列前（聊天管理器.分发消息） |
| `on_pre_plan` | `(聊天流, 消息, 对话历史)` | 可修改对话历史 | 修改型 | 规划模型调用前 |
| `on_post_plan` | `(聊天流, 工具调用列表)` | 可修改工具调用列表 | 修改型 | 规划输出后、工具执行前 |
| `on_post_tool` | `(结果)` | 可修改工具结果 | 修改型 | 单个工具执行后 |
| `on_pre_reply` | `(聊天流, system提示词, user提示词)` | 可修改提示词 | 修改型 | 回复模型调用前 |
| `on_post_reply` | `(回复内容)` | 可修改回复内容 | 修改型 | 回复模型输出后、发送前 |
| `on_send_message` | `(回复数据)` | `回复数据` 或 `None`(拦截) | 过滤型 | 消息发送到适配器前 |

### 加载流程
1. `main.py` 启动 → 初始化 `插件管理器` 实例
2. 扫描 `插件/` 目录下所有子目录
3. 读取每个子目录的 `plugin.toml`，验证元信息
4. 合并 `插件配置/<插件名>.toml` 覆写配置
5. 导入 `main.py`，调用 `入口(上下文)` 函数
6. 插件通过上下文注册钩子和工具
7. 按 priority 排序钩子注册表
8. 触发所有 `on_load` 钩子
9. 错误隔离：单个插件失败不影响其他插件和主流程
